The frame

Most AI consultants sell a new model. The useful work is deciding where the model belongs.

Most companies shopping for an AI consultant assume the answer is a new model, a new platform, or a new license. That is usually wrong. The right frame is a simple three-way split of any ops problem. The useful job of an outside consultant is figuring out which layer each piece of a problem belongs on.

60%
Traditional code

Databases, services, scripts. The work Postgres and a cron job handle faster, cheaper, and with better guarantees than any LLM.

30%
Rule-based logic

Deterministic routing, thresholds, policy trees. Cheap, auditable, testable. The layer most MSSPs under-invest in.

10%
Genuine AI work

Alert correlation, unstructured-to-structured transforms, context retrieval. The pieces that are actually model-shaped.

Applied to the MSSP operation

Managed firewall, WAF, IPS, SIEM, email protection, vulnerability management. Each one sits on a different layer.

For an MSSP operation that runs those services across multiple clients, the mapping is specific. It is also the part that decides whether per-client margin holds as the account list grows.

AI-shaped

Alert correlation and context retrieval

Pull the right per-client baseline, history, and similar incidents into an analyst's view at the moment of triage. Reduces the time from alert to first hypothesis.

Rule-shaped

Triage routing and escalation

Deterministic policy. Which client, which severity, which analyst, which SLA clock, which on-call tree. Rules belong here, not a model.

Document-shaped

Evidence packaging for compliance

SOC 2, HIPAA, CMMC. Mostly structured-document work. Compresses hard when per-client context is organized rather than scattered across tools.

Proof of shape

A compliance platform built for a different regulated operator, similar shape to the MSSP evidence workflow.

Not identical. Not a drop-in. The same shape.

160+ hours → under 5 minutes Per compliance cycle

Eduba built a multi-agent compliance platform for VigilOre, an operator in the DRC mining supply chain. The write-up and attestation work that used to take their team 160+ hours per cycle now runs in under 5 minutes. MSSP evidence packaging (SOC 2, HIPAA, CMMC, vulnerability-scan reports, incident logs) is the same structured-document problem in a different vertical.

The methodology behind that build is published. Interpretable Context Methodology: Folder Structure as Agent Architecture, submitted to ACM TiiS. The idea is that client context can be organized as a layered filesystem with measurable interpretability gains. That is exactly how an MSSP should store per-client configuration, escalation trees, and compliance evidence so that any analyst on the team can pick up any client without losing the prior context.

A small U.S.-based team running like a large SOC without hiring for it. That is the useful outcome.
